# Technical — 기술적 분석 표준 절차

기술적 애널리스트가 일관된 차트 분석을 수행하기 위한 표준 지표·해석 가이드.

## 1. 분석 기간 표준

| 봉 차트 | 기간 | 목적 |
|--------|------|------|
| 일봉 | 1년 (252거래일) | 중기 추세 |
| 일봉 | 3개월 | 단기 모멘텀 |
| 주봉 | 3~5년 | 장기 추세 / 사이클 |
| 월봉 | 10년 이상 | 슈퍼사이클 / 박스권 |

분석 시 1년 일봉을 기본으로 하고, 추가 관점이 필요할 때 다른 시간대를 활용한다.

## 2. 이동평균선 정렬

표준 MA: **5 / 20 / 60 / 120 / 200일**

| 정렬 | 의미 |
|------|------|
| 정배열 (5 > 20 > 60 > 120 > 200) | 강한 상승 추세 |
| 역배열 (5 < 20 < 60 < 120 < 200) | 강한 하락 추세 |
| 혼조 | 추세 전환 또는 박스권 |

**골든크로스**: 단기선이 장기선을 상향 돌파. 일반적으로 60일선↑200일선이 의미 있는 시그널.
**데드크로스**: 반대.

## 3. 모멘텀 지표 산출 및 해석

### RSI(14)
- 과매수: 70 이상
- 과매도: 30 이하
- **다이버전스**: 주가가 신고가인데 RSI가 신고가에 도달하지 못함 → 약세 시그널
- 다이버전스는 시간차가 있어 단독 매매 시그널로는 약함, 다른 지표와 결합

### MACD (12, 26, 9)
- MACD = EMA(12) - EMA(26)
- Signal = EMA(MACD, 9)
- Histogram = MACD - Signal
- 시그널: MACD가 Signal을 상향/하향 돌파, 0선 돌파, 히스토그램 부호 변경

### Stochastic (14, 3, 3)
- %K와 %D의 교차 + 과매수/과매도(80/20) 영역
- RSI 대비 단기 신호에 민감

## 4. 변동성 / 밴드

### 볼린저밴드 (20, 2σ)
- 상단: 20MA + 2σ
- 하단: 20MA - 2σ
- 상단 터치 → 단기 과열 가능성
- 밴드 폭이 좁아진 후 확장 → 추세 전환 가능성
- 단순한 "상단=매도, 하단=매수" 해석은 추세장에서 위험

### ATR (14)
- 평균 진폭으로 변동성 측정
- 스탑로스 폭 결정에 사용 (예: 진입가 - 2 × ATR)

## 5. 거래량 분석

- **평균 거래량 대비** 최근 거래량 (5일 평균 vs 60일 평균)
- **가격-거래량 다이버전스** — 가격이 오르는데 거래량이 줄어드는 경우 추세 약화 가능성
- **OBV** (On-Balance Volume) — 종가 기준 거래량 누적, 추세 검증
- 거래량 급증 구간은 `sentiment-analyst`에게 해당 시점의 이벤트 조사 요청 (SendMessage)

## 6. 지지·저항 식별

식별 방법:
1. **최근 1~2년의 의미 있는 고점/저점** — 매물대
2. **이동평균선** — 60/120/200일선이 자주 지지·저항 역할
3. **심리적 라인** — 한국에서는 ₩50,000 / ₩70,000 / ₩100,000 (천원, 만원, 십만원 단위)
4. **갭(Gap) 영역** — 급등/급락 시 발생한 미체결 갭은 향후 채워질 영역

표 형식으로 정리:

| 종류 | 가격 영역 | 근거 | 강도 |
|------|---------|------|------|
| 저항 | ₩X | 1년 고점, 거래량 집중 | 강 |
| 저항 | ₩Y | 120일선 | 중 |
| 지지 | ₩Z | 60일선 + 매물대 | 강 |

## 7. 시나리오 작성

3가지 시나리오를 표준화:

### Bullish
- **트리거**: ${조건 1}, ${조건 2}
- **타깃**: ₩A → ₩B (1차 저항 → 2차 저항)
- **무효화 조건**: ${가격이 X 아래로 종가 마감}

### Base
- **가정**: 박스권 또는 완만한 추세
- **상하단**: ₩C ~ ₩D

### Bearish
- **트리거**: ${조건}
- **하방 타깃**: ₩E → ₩F
- **무효화 조건**: ${가격이 Y 이상으로 종가 마감}

각 시나리오는 **확률**을 명시하지 않거나, 명시할 경우 객관적 근거(과거 유사 패턴 발생 빈도 등)를 함께 제시한다.

## 8. 기술적 분석의 한계 (보고서에 명시)

- 기술적 분석은 후행성을 갖는다 — 추세 형성 후 인식
- 자기실현적 예언 효과는 패턴이 널리 알려질수록 약해진다
- 매크로/이벤트 충격에는 무력하다
- 단독 의사결정 도구가 아니라 펀더멘털과 함께 사용해야 한다

이 한계 섹션은 모든 기술적 분석 보고서에 포함한다.
